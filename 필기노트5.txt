이미지를 웹 페이지에 보여줄 때는 몇 가지 문제가 있음
해상도가 높은 기기와 해상도가 낮은 기기에서 이미지의 처리에 문제가 있음
해상도가 높은 기기를 기준으로 이미지를 준비하면 이미지 파일의 크기가 커지고
해상도가 낮은 기기를 기준으로 이미지를 준비하면 이미지 파일의 크기는 작아지지만 고해상도 기기에서는 이미지가 깨지거나 작게 보이는 문제가 있음
모바일 기기에서는 기기의 화면을 가로 / 세로 전환할 수 있기 때문에 이러한 부분도 고려를 한다면 더 많은 문제가 생김
<img src="기본 이미지" srcset="파일[, 파일2, ...]">
srcset(픽셀너비서술자) -> 해상도에 따라서 다른 파일 보여줌

textalign -> 수직 가운데 정렬
lineheight -> 수평 가운데 정렬
texttransform: uppercase; -> 대문자로

Component(컴포넌트)
- 공통적으로 사용되는 디자인 요소
- 컴포넌트를 생성하면 다른 페이지나 다른 디자인에서 컴포넌트를 불러다 사용할 수 있음
- 헤더, 푸터 
Instance(인스턴스)
- 컴포넌트를 불러다 사용하게 되면 사용한 그 요소를 인스턴스라고 함

Component -> 컴포넌트
컴포넌트를 생성하면 피그마 전반에서 다시 사용 가능한 요소를 생성하는 것
컴포넌트는 마스터 컴포넌트 / 컴포넌트 인스턴스가 있음
마스터 컴포넌트 -> 다시 사용 가능한 요소를 생성하는 것
컴포넌트 인스턴스 -> 마스터 컴포넌트를 불러와서 사용한 것
컴포넌트 인스턴스와 마스터 컴포넌트는 연결이 되어있음
따라서 마스터 컴포넌트를 수정하면 컴포넌트 인스턴스들도 모두 다 수정됨
마스터 컴포넌트를 등록하면 같은 팀으로 등록된 사람들도 모두 다 같이 사용할 수 있음
